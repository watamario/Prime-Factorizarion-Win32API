・このプログラムについて

　入力された自然数を素因数分解するだけのシンプルなプログラムです。
　しかし、その裏には計算中の"応答なし"回避のためのマルチスレッド化や計算結果の
　エディットボックス表示その他のWindows APIを用いたC++のGUIプログラミングにおける
　基礎的な内容をたくさん詰め込んでいます。
　DXライブラリを使えばもっと簡単に作れるし、時代はUWPアプリへと移行が進んでいますが、
　Windows APIに挑戦してみたいという私のような変わり者はぜひ参考にしてみてください。


・使用上の注意

　watamarioは、このソフトウェアの使用によって発生した計算結果の誤り、バグ等による
　誤動作を含むいかなる損害に対しても補償致しません。自己責任でご利用ください。

　これをテンプレートとして作成した自作プログラムの配布に関しては、私が著作権を
　主張することはございませんので、自由に行って頂いて構いません。


・ビルド時の要件

　このプロジェクトは、Visual Studio Enterprise 2019で作成されています。
　また、動作環境を広げるためにWindows XPがサポートされるビルドセットを使用しています。
　このビルドセットは最初に手動インストールを行わないと使用できません。
　インストールされていなければ開いたときに求められるので、"不足しているプラグインの
　インストール"のような旨の項目をクリックしてインストールしてください。

　インストールできない環境でも、以下の手順でWindows XPサポートを解除すれば大丈夫です。

　　デバッグ->(プロジェクト名)のプロパティ->構成プロパティ を表示させ、上部の"プラット
　　フォーム"を"すべてのプラットフォーム"に切り替える。その上で、以下の項目を変更する。

　　1. 全般->プラットフォーム ツールセット を "Visual Studio 2019 (v142)" に変更※
　　2. 一旦OKを押して閉じ、再度同じ項目を表示させる。
　　3. 全般->Windows SDK バージョン を "10.0 (最新のインストールされているバージョン)"
　　　 に変更※
　　4. C/C++->言語->準拠モード を"はい"に変更

　※2020年8月現在のVisual Studio 2019における設定です。より新しいバージョンでも、
　　表示されたオプションの中で最新のものを選択すれば大丈夫なはずです。


・注釈

　ソースコードは、firsttestフォルダの1.cppです。
　実行可能バイナリは、x86(32bit)はWin32、x86(64bit)はx64、ARM(32bit)版はARM、
　ARM(64bit)版はARM64フォルダの、デバッグビルドはDebugフォルダに、リリースビルドは
　ReleaseフォルダにPF_(対象CPUアーキテクチャ名).exeとして生成されます。
